IP Overview
-----------

A register file design for a RISC-V core. Each word is divided into fixed-width
chunks. A global chunk selector is used to restrict all read and write operations
to one chunk at a time. Register 0 is hard-wired to zero on all read ports to
follow the RISC-V specification. One synchronous write port and a configurable
number of combinational read ports are provided.

== Design Requirements

[cols="1,4"]
|===
| Requirement ID | Description

| `DR-REGFILE-010`
| The module *MUST* store `NUM_WORDS` words, each `WORD_WIDTH` bits wide.

| `DR-REGFILE-020`
| Chunk stem:[k] of word stem:[n] *SHALL* correspond to bits
stem:[[(k+1) \cdot \texttt{CHUNK\_WIDTH} - 1 : k \cdot \texttt{CHUNK\_WIDTH}]]
of register stem:[n], partitioning each word into `NUM_CHUNKS` non-overlapping
chunks each `CHUNK_WIDTH` bits wide.

| `DR-REGFILE-030`
| Register contents *MUST* be zero at most one clock cycle after the synchronous
reset (`rst_ni`) is asserted.

| `DR-REGFILE-040`
| Writes to the register file *MUST* only occur on the rising edge of `clk_i`.

| `DR-REGFILE-041`
| Writes to the register file *MUST* only occur when `wen_i` is asserted.

| `DR-REGFILE-042`
| Contents of `wdata_i` *MUST* be written only to the chunk selected by
`chunk_sel_i` within the register selected by `waddr_i`. All other chunks of
the target register *SHALL* remain unchanged.

| `DR-REGFILE-043`
| Writes to register 0 *SHALL* be silently ignored, regardless of the state of
`wen_i`.

| `DR-REGFILE-050`
| The read output `rdata_o[i]` *SHALL* be combinationally derived from the
current register state, `raddr_i[i]`, and `chunk_sel_i`. There *SHALL NOT* be
any latency on reads.

| `DR-REGFILE-051`
| The read output `rdata_o[i]` *MUST* reflect the chunk selected by `chunk_sel_i`
within the register selected by `raddr_i[i]`.

| `DR-REGFILE-052`
| The read output `rdata_o[i]` *MUST* be zero when `raddr_i[i]` is zero,
regardless of any prior writes to register 0.

| `DR-REGFILE-053`
| Requirement `DR-REGFILE-052` *SHALL* be unconditional: it *MUST* hold at
power-on without reset having been asserted, and at all times during normal
operation.

| `DR-REGFILE-060`
| The module *SHALL NOT* implement any bypass logic. A read and write to the same
address in the same cycle *SHALL* return the value held in the register prior to
the rising clock edge of the current cycle.

Architecture
------------

The module is composed of a flop array, a combinational read network, and a clocked write port.

=== Storage

The register array is declared as a packed 2D structure:

[source]
----
logic [NUM_WORDS-1:0][WORD_WIDTH-1:0]
----

By default, this will be a stem:[16 \times 32] bit array.

=== Chunk Addressing

The active chunk within a word is addressed by `chunk_sel_i` using a part-select:

[source]
----
register[addr][chunk_sel_i * CHUNK_WIDTH +: CHUNK_WIDTH]
----

The input `chunk_sel_i` is a global selector; the write port and all read ports
will share the same active chunk.

=== Read Ports

The reads are all combinational. For a read port stem:[i]:

[stem]
++++
\texttt{rdata\_o}[i] =
\begin{cases}
	0 & \text{if} \; \texttt{raddr\_i}[i] = 0, \\
	\texttt{register}[\texttt{raddr\_i}[i]][\text{chunk}] & \text{otherwise}.
\end{cases}
++++

There is no read-after-write forwarding; a write is not visible on a read port
until the next cycle.

=== Write Port

Write to the registers are clocked on the rising edge of `clk_i`.
Both of the following conditions must be satisfied for a write to occur:

. `wen_i` is asserted; and
. `waddr_i` is not zero, as register 0 is write-protected.

Timing
------

[cols="1,2"]
|===
| Operation | Latency

| Read
| 0 cycles (combinational)

| Write visible on read port
| 1 cycle

| Same address back-to-back writes
| Safe
|===

No reset is implemented; register contents are undefined on power-on until
written to.

Design Notes
------------

The following documents implementation decisions, including intentional deviations
from standard practice and known limitations.

* The register file is implemented with a chunk selector to match the width of
the datapath design.

* There are no assertions to guard the `WORD_WIDTH % CHUNK_WIDTH == 0` constraint;
violating it will leave the top bits of each word unreachable.

* The write block contains a `dff-name-style` waiver to suppress a
naming convention warning from the `verible-verilog-lint` linter; the decision
was to have the array named `register`, which deviates from the project's
style guidelines.
